DHCP 
Multiple Choice
14/14 points
1. Что такое DHCP?
Cетевой протокол, позволяющий компьютерам автоматически получать IP-адрес и другие сетевые параметры, необходимые для работы в сети TCP/IP.

2. DHCP имеет:
клиент-серверную архитектуру

3. По какому типу запроса происходит получения адреса по DHCP?
broadcast

4. Типы DHCP запросов:
DHCPDISCOVER
DHCPOFFER
DHCPACK
DHCPREQUEST

5. Выберите правильный порядок запросов от клиента к DHCP серверу при запросе IP-адреса по DHCP
DHCPDISCOVER - DHCPOFFER - DHCPREQUEST - DHCPACK

6. Какие основные сетевые настройки можно получить по DHCP?
IP-аддрес
Маску подсети
Шлюз по умолчанию
Адрес DNS сервера

7. Какой IP адрес в заголовке пакета использует компьютер при запросе IP адреса по DHCP?
0.0.0.0

8. На какой IP адрес происходит запрос получения IP адреса по DHCP?
255.255.255.255

9. Перечислите преимущества использования DHCP
DHCP минимизирует ошибки конфигурации, вызванные ручной настройкой IP-адреса, такие как опечатки или конфликты адресов.
Централизованная и автоматическая конфигурация TCP/IP для всех клиентов в сети.
Пересылка исходных сообщений DHCP с использованием агента ретрансляции DHCP, что устраняет необходимость в DHCP-сервере в каждой подсети.

10. Какой протокол и порты используются в DHCP?
UDP 67/68

11. Что использует DHCP сервер для резервирования IP-адреса для клиента?
MAC-адрес клиентского компьютера

12. Какая функция используется для доставки DHCP запроса к DHCP серверу, который находится в другой сети?
DHCP RELAY

13. Когда компьютер отправляет запросы DHCP и не получает ответы, из какой подсети компьютер автоматически назначает себе IP-адрес?
169.254.0.0/16 (APIPA)

14. Какая команда используется для повторного запроса и отказа от настроек, полученных по DHCP в Windows?
ipconfig /release, ipconfig /renew
